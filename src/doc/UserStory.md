明白了！您需要的不是整个HR系统，而是其中的一个**独立、可嵌入的“产假测算模块”**。

这个模块的核心目标是：**根据中国现行法规，输入员工基本信息后，自动、准确地计算出该员工应享有的产假总天数和生育津贴/工资待遇。**

---

### **产假测算模块设计（面向HR使用）**

#### **一、 输入参数（用户需填写）**

1.  **员工基本信息：**
    *   员工姓名/工号
    *   **参保所在地（省、市）** 👉 *这是最关键的数据，决定适用哪个地方政策*
    *   预产期（或预产期证明日期）
    *   生育情况：单胎 / 多胞胎（双胞胎、三胞胎...） / 是否难产（需医生证明）
    *   是否为晚育？（*注：目前多数省份已取消晚育假，但个别地区或有特殊规定，可作为可选字段*）
    *   是否已连续缴纳生育保险满12个月？（是/否）👉 *决定能否领取生育津贴*

2.  **薪酬信息（用于津贴/工资计算）：**
    *   员工产假前12个月的平均工资（系统可对接薪资数据自动获取，或手动输入）
    *   用人单位上年度职工月平均工资（可对接社保数据或手动输入）

---

#### **二、 核心计算逻辑（系统自动执行）**

##### **1. 产假总天数计算**

```
基础产假：98天（国家规定）
+
地方奖励假：X天（根据参保地自动匹配）
+
特殊情况加假：
  - 难产：+15天
  - 多胞胎：每多生育1个婴儿 +15天
=
**应休产假总天数**
```

**示例：**
*   员工在北京参保，单胎顺产：
    *   98天（国家） + 60天（北京奖励假） = **158天**
*   员工在广东参保，双胞胎难产：
    *   98天 + 80天（广东奖励假） + 15天（难产） + 15天（多胞胎） = **208天**

> 📌 **系统内置政策库：** 需维护全国各省市的奖励假天数表，并支持自动更新。

##### **2. 生育津贴与工资待遇计算**

根据员工是否满足生育保险领取条件，分两种情况：

**情况A：已参保且满足条件（可领取生育津贴）**

```
生育津贴金额 = 用人单位上年度职工月平均工资 ÷ 30 × 产假总天数
员工应得工资总额 = 员工产假前12个月平均工资 ÷ 30 × 产假总天数

企业需补足差额 = 
  IF (员工平均工资 > 用人单位平均工资) THEN 
      (员工平均工资 - 用人单位平均工资) ÷ 30 × 产假总天数
  ELSE 
      0
```

**最终员工实得：生育津贴 + 企业补差**

**情况B：未参保或不满足条件（企业全额承担）**

```
企业应支付工资总额 = 员工产假前12个月平均工资 ÷ 30 × 产假总天数
```

> 📌 **关键逻辑：** 系统自动判断，并清晰展示计算过程。
